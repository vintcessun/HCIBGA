# 七彩信息导入模块 API 文档

## 模块说明
提供七彩信息文件导入功能，负责文件上传、校验与导入提交。

---

## 文件服务接口

### 1. 检查文件是否存在
**路径**：`GET /api/files/check`  
**方法**：GET  
**请求参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| md5 | string | 是 | 文件 MD5 值 |
| filename | string | 是 | 文件名 |

**响应示例**
```json
{
  "exists": false,
  "file_id": null
}
```

---

### 2. 上传导入文件
**路径**：`POST /api/files/upload`  
**方法**：POST  
**请求体**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | binary | 是 | 要导入的七彩信息文件 |

**响应示例**
```json
{
  "file_id": "uuid",
  "url": "https://cdn.example.com/files/uuid",
  "md5": "string"
}
```

---

## 导入处理接口

### 3. 提交导入任务
**路径**：`POST /api/imports/qicai`  
**方法**：POST  
**请求体**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| files | array | 是 | 已上传文件 ID 列表，如 ["uuid1","uuid2"] |

**响应示例**
```json
{
  "success": true,
  "message": "导入任务已提交"
}
```

---

## 错误码
| 状态码 | 说明 |
|--------|------|
| 400 | 文件格式错误 |
| 413 | 文件过大 |
| 500 | 导入失败 |

---

## 模块交互图
```
用户上传文件
   ↓
检查文件是否存在
   ↓
上传文件（如不存在）
   ↓
点击提交导入
   ↓
后端执行导入任务
   ↓
返回任务结果
