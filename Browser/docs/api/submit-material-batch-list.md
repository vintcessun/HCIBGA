# 提交材料管理 - 批次列表接口文档

## 一、模块简介
该模块用于展示及管理提交材料的批次信息。  
对应前端页面：`/submit-material/batch-list`  
组件路径：`src/views/user/batch-list/manage-batch.vue`  

当前版本展示静态数据（模拟两条批次信息），后续将对接后端接口获取动态批次列表。

---

## 二、接口概览

| 接口功能 | 请求方法 | 请求路径 |
|-----------|------------|-----------|
| 获取批次列表 | GET | `/api/submit-material/batch-list` |

---

## 三、请求参数

### 1. Query 参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| page | number | 否 | 当前页码 | 1 |
| size | number | 否 | 每页条数 | 10 |
| status | string | 否 | 批次状态过滤，可选值：`进行中`、`已完成` | 进行中 |

---

## 四、返回参数

### 1. 返回示例

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "batch": "批次1",
      "total": 10,
      "status": "进行中"
    },
    {
      "batch": "批次2",
      "total": 15,
      "status": "已完成"
    }
  ],
  "timestamp": 1730648400000
}
```

### 2. 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| batch | string | 批次名称 |
| total | number | 提交总数 |
| status | string | 当前批次状态（进行中/已完成） |
| timestamp | number | 响应时间戳（毫秒） |

---

## 五、错误码规范

| 错误码 | 含义 | 解决建议 |
|--------|------|-----------|
| 0 | 请求成功 | — |
| 400 | 参数错误 | 检查 Query 参数是否完整 |
| 401 | 未授权 | 登录状态已过期，重新认证 |
| 403 | 无访问权限 | 联系管理员开通权限 |
| 500 | 服务器错误 | 稍后重试或联系后台维护人员 |

---

## 六、时间戳规范
后端返回 `timestamp` 字段记录响应生成时间，精确到毫秒。  
推荐格式化显示为：`YYYY-MM-DD HH:mm:ss`。

---

## 七、备注
- 当前前端静态展示数据结构与后续后端接口返回保持一致。
- 接口结构建议遵循 `successResponseWrap` 格式规范。  
- 如后续增加分页、筛选功能，可在 Query 参数中新增字段并在表格组件中绑定。
